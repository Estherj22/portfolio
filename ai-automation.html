How it works

Automation
Architecture

Seven stages from raw input to structured output. Each stage is explicit and auditable — this is the architecture you'd present to a CTO or technical co-founder.

01
Trigger — monthly ops cycle ends
The ops lead decides it's time to send an investor update. This can be calendar-triggered (scheduled automation) or manual (on-demand, as in this prototype).
Trigger layer
02
Input — three data streams collected
Operational metrics (revenue, units, pipeline), narrative notes (wins, blockers, asks), and configuration parameters (tone, period, audience) are collected as structured inputs.
Input layer
03
Prompt constructor — context injection
A structured prompt template injects the user's raw notes alongside explicit format rules: lead with the strongest metric, acknowledge challenges, end with one clear ask, stay under 300 words.
Processing layer
04
Input validation — completeness check
Before calling the API, the system checks that the input contains enough information to generate a useful update. Incomplete inputs prompt the user to fill gaps rather than producing a poor output.
Processing layer
05
Claude API — claude-sonnet-4 generates the draft
The engineered prompt is sent to Anthropic's Claude API. The model generates a subject line and full email body, structured according to the format rules embedded in the prompt. Average latency: 3–6 seconds.
AI layer · claude-sonnet-4
06
Response parser — splits subject, body, metadata
The raw API response is parsed to extract the subject line separately from the email body, and to capture metadata (word count, tone used, generation time) for the analytics strip.
AI layer
07
Outputs — draft email, log, analytics
Three outputs are produced: the draft email (ready to review and send), an update log entry for the audit trail, and analytics data (tone, word count, generation time) feeding the ops dashboard.
Output layer
